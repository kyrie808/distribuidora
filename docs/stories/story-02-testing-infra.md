# Story 02: Setup Testing Framework

## Description
As a developer, I want to have automated tests (Unit and E2E) so that I can refactor code and add features without introducing breaking changes.

## Tasks
- [ ] Install and configure **Vitest** for unit/hook testing.
- [ ] Install and configure **Playwright** for E2E testing.
- [ ] Create a test for a critical utility (e.g., `formatCurrency`).
- [ ] Create a simple E2E test for the Login flow.

## Acceptance Criteria
- [ ] `npm run test:unit` executes correctly.
- [ ] `npm run test:e2e` opens the browser and validates UI components.
- [ ] Testing documentation is added to the codebase.
